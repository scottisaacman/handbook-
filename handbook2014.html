Installing GhostBSD

Minimum System Requirements:

LXDE spin

  • Pentium II
  • 256 MB RAM
  • 10GB of free hard drive space
  • Network card
  • Sound card (if you want sound)

Gnome spin

  • Pentium III
  • 256 MB RAM
  • 10GB of free hard drive space
  • Network card
  • Sound card (if you want sound)

Recommended Base Systems

LXDE spin

  • Pentium III
  • 512 MB RAM
  • 20GB of free hard drive space
  • Network card
  • video card (more fun with games)
  • Sound card (if you want sound)

Gnome spin

  • Pentium 4
  • 1024 MB RAM
  • 10GB of free hard drive space
  • Network card
  • video card (more fun with games)
  • Sound card (if you want sound)

Supported Processors

GhostBSD works on 32-bit(i386) and 64-bit processors. These are manufactured by AMD and Intel.

Video Cards

GhostBSD uses X.org drivers for graphics support and supports many video cards. 3D acceleration works on most video cards (NVIDIA, ATI, Intel). If 3D does not work, you may need to choose the 2D driver. If that fails, use the VESA driver.

Hardware

Although most hardware works with GhostBSD, there are exceptions. Please report the problem so that the developers address it. Remember, GhostBSD is really FreeBSD, so any hardware that works on FreeBSD will work on GhostBSD. If you know you are using unique hardware, before installing check the FreeBSD 9.x hardware notes.

Wireless Devices

Wireless device drivers are a special problem. GhostBSDs supports some wireless PCMCIA, USB, and other networking devices. A few manufacturers provide drivers for Linux/Unix. If you are about to purchase a device for use, please check the Free BSD site.

Pre-installation Tasks

Back-up Data

If the hard drive has important data on it, copy the data to another storage device before starting your installation.

Check for FreeBSD Errata

GhostBSD is based on FreeBSD. Although the FreeBSD Project strives to ensure that each release of FreeBSD is as stable as possible, bugs occasionally creep into the process. On very rare occasions those bugs affect the installation process. As these problems are discovered and fixed, they are noted in the FreeBSD Errata on the FreeBSD web site. Check the errata before installing to make sure that there are no problems that might affect the installation.

Prepare the Installation Media

The installation system for GhostBSD can be downloaded for free. GhostBSD is available in .iso (DVD) or .img (USB stick and flash drive) file extension. Copies of GhostBSD installation media are available at the GhostBSD download page.

Creating a bootable Memory Stick in FreeBSD, PC-BSD or GhostBSD.

Warning: The example below shows /dev/da0 as the target device where the image will be written. Be very careful that the correct device is used as the output target, or you may destroy a drive other than the USB drive and lose data. The memory stick image has a .img extension. The .img file is not a regular file. It is an image of the complete contents of the memory stick. It cannot simply be copied like a regular file, but must be written directly to the target device with dd(1): dd if=/path/to/GhostBSD-2.5-i386.img of=/dev/da0 bs=1mM

Creating a bootable Memory Stick in GNU/Linux.

Warning: The example below shows /dev/sdf as the target device where the image will be written. Be very careful that the correct device is used as the output target, or you may destroy existing data. The memory stick image has a .img extension. The .img file is not a regular file. It is an image of the complete contents of the memory stick. It cannot simply be copied like a regular file, but must be written directly to the target device with dd(1): sudo dd if=GhostBSD-2.5-i386.img of=/dev/sdf bs=1M

Creating a bootable Memory Stick in Windows

Warning: Be sure to give the correct drive letter as the output target, or you may overwrite and destroy existing data. Windows Image Writer for Windows is a free application that can write an image files to a memory sticks. Download it and extract it. Double-click the Disk Imager icon to start the program. Verify that the drive letter shown under Device is the drive with the memory stick. Click the folder icon and select the image to be written to the memory stick. Click Save to accept the image file name. Verify that everything is correct, and that no folders on the memory stick are open in other windows. When everything is ready, click Write to write the image file to the memory stick.

Starting GhostBSD Live Media

The computer BIOS contains boot sequence instructions. If you are going to boot from the DVD or a USB device, then you must cue them before the computer's hard drive boot. Typically the BIOS boot sequence instructions can be accessed by pressing F10, F11, F12, or Escape and select SETUP. Usually the computer provides instructions during the start up sequence. Browse through the menu options to find Boot Sequence and que appropriately. Some computers allow selection of boot devices without accessing the BIOS setup. Again, this is usually presented on the computer screen extremely briefly during the startup sequence.

When using a bootable USB stick, plug in your USB stick before turning on the computer. If you are booting from DVD drive, then you will need to turn on the computer, and insert the DVD at the first opportunity. If you are too slow, the boot sequence may miss reading the DVD.

Old Computers

If you have a very old computer and the BIOS does not support booting from the desired media, Plop Boot Manager can be used to boot DVD or USB media.

Installing with GBI.

GBI is a GTK graphical BSD installer program written by Eric Turgeon and introduced in 2011 for GhostBSD 2.5. GBI use the pc-sysinstall back-end developed by Kris More of PC-BSD.

Language Menu

Selecting language menu

Select the language you will like to use after the installation. If your language is not show in selection, you can change it at the first boot to the GDM login. In compliance with Canadian practice, GBI will be multilingual in the future and display the languages the developers speak.

Keyboard Menu

Selecting keyboard model, layout and variant menu

The model option is the model of your keyboard. The list is large. You do not need to specify one model. But using this option it will give you full support of your keyboard. The layout option is your country specific keyboard layout. The variant option is your type of keyboard. This option give you the opportunity to select Dvorak, Qwerty and more. If you use the default GhostBSD system keyboard setup you can skip by just click on on forward. Also if you don't know your keyboard model and variant, skip forward.

Time Zone menu

Setting the time zone for your machine allows it to automatically correct for any regional time changes and perform other time zone related functions properly.

Selecting Time Zone:

Select the continent you live in and chose your city or a listed city that is in your time zone.

Disk and Installation Option Menu

Selecting from Multiple Disks

If multiple disks are connected, choose the one where GhostBSD is to be installed.

Selecting Installation option

A portion of the disk or the entire disk can be allocated to GhostBSD. If "Use Entire Disk" is chosen, a general partition layout filling the whole disk is created. By selecting "Auto label Partition", you can creates a partition slices without labeling and allocating space. By selecting "Customize Disk Partition",the advanced user can create custom partition slices and create multiple labels.

Boot Loader

GhostBSD gives you the option of using the BSD Boot Loader or not. If you dual boot with GNU/Linux, you should use the Gnu/Linux Grub boot loader. If you dual boot GhostBSD with MS Windows, you should use the BSD boot loader.

Selecting Use Entire Disk

When using this option it use all the space of the hard drive selected.
